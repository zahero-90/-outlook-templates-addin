📧 My Email Templates — Outlook Web Add-in

A proper Office COM-class add-in built using the official Microsoft Office Web Add-in standard. Deploys exactly like Adobe Sign, DocuSign, or Nitro PDF — as a registered add-in with a ribbon button, manageable from Outlook's Add-ins settings.

✓ Office Web Add-in Standard ✓ Ribbon Button (Compose + Reply) ✓ M365 Admin Center Deployment ✓ Sideload Per User ✓ Signature Preserved ✓ Unlimited Templates

⚖️ COM Add-in vs Office Web Add-in — What's the Difference?

🔧 COM Add-in (VSTO)
e.g. Adobe Acrobat PDF Maker, older Nitro
  • Deep Windows integration
  • Accesses local file system
  • Requires .NET / Visual Studio to build
  • Must be installed per machine (MSI/EXE)
  • Cannot be deployed via M365 Admin Center
  • Breaks on major Outlook updates
  • Desktop Outlook only
🌐 Office Web Add-in RECOMMENDED
e.g. Adobe Sign, DocuSign, Nitro (new versions)
  • Works on Desktop, Web, Mobile
  • Deploy to ALL users via M365 Admin Center
  • Appears in Outlook Add-ins list (enable/disable)
  • Ribbon button — same as Adobe/DocuSign
  • Microsoft-supported standard (future-proof)
  • No machine-level installation needed
  • Requires HTTPS hosting for the add-in files
ℹ️ This add-in is built as an Office Web Add-in — the same technology that Adobe Sign, DocuSign, Grammarly, and the new Nitro use. It is the current Microsoft standard for all new Outlook integrations.

1 Host the Add-in Files (Requires HTTPS)

The add-in files must be reachable over HTTPS. Choose the option that suits your environment:

GitHub Pages FREE
Easiest option. Free HTTPS hosting via GitHub. Ideal for quick deployment.
Azure Static Web Apps FREE TIER
Microsoft's own hosting. Free tier, global CDN, connects to GitHub for auto-deploy.
SharePoint / SPO
Host the files in a SharePoint document library. Stays inside your M365 tenant.

Option A — GitHub Pages (Recommended, Free, 5 minutes)
  1. Create a free account at github.com (or use an existing one)
  2. Click + New repository → Name it my-templates-addin → Set to Public → Create
  3. Upload all files from this package, keeping the folder structure: 
    my-templates-addin/
├── manifest.xml
├── src/
│   ├── taskpane.html / taskpane.js / taskpane.css / commands.html
└── assets/
    └── icon-16.png  icon-32.png  icon-64.png  icon-80.png
  4. Go to repository Settings → Pages → Source: Deploy from branch → main → / (root) → Save
  5. Your hosting URL will be: https://YOUR-USERNAME.github.io/my-templates-addin

Option B — Azure Static Web Apps
  1. Go to portal.azure.com → Create resource → Static Web App
  2. Connect to your GitHub repo, choose branch main
  3. Azure auto-deploys and gives you an HTTPS URL like https://agreeable-beach-12345.azurestaticapps.net

2 Update manifest.xml with Your URL

Open manifest.xml in Notepad and replace every occurrence of https://YOUR-DOMAIN.com/addin with your actual URL.

Find → Replace All: 
Find:    https://YOUR-DOMAIN.com/addin
Replace: https://YOUR-USERNAME.github.io/my-templates-addin
         (or your Azure / SharePoint URL)
⚠️ There are 8 places in the manifest that reference this URL. Make sure they are all updated before proceeding.
⚠️ The URL must be HTTPS. HTTP will not work. Outlook enforces this for all add-in files.

3A Deploy to All Users via M365 Admin Center

This deploys the add-in to your entire tenant automatically. Users do not need to do anything.

⚙️ Microsoft 365 Admin Center — Integrated Apps
📧

Priority Worldwide IT · Outlook · All users
Deployed
  1. Go to admin.microsoft.com and sign in as a Global Admin
  2. In the left menu: Settings → Integrated apps
  3. Click Upload custom apps
  4. Choose Office Add-in as the app type
  5. Select "Upload manifest file (.xml) from device"
  6. Browse and select your updated manifest.xml
  7. Click Next
  8. Under Assign users, choose:
    • Entire organization — deploys to all users ✓
    • or specific users/groups
  9. Click Next → Finish deployment
  10. The add-in appears in Outlook for all assigned users within 24 hours (usually within minutes)
✅ Users will see My Templates in their Outlook ribbon automatically — no action required from them. It appears in Get Add-ins → Admin-managed and is shown as a pinned ribbon button.

3B Install Per User (Sideload — No Admin Required)

Each user can install the add-in themselves from their Outlook. No IT/admin access needed.

Outlook Desktop (Windows):
  1. Open Outlook → Click Get Add-ins (in the Home ribbon, or File → Manage Add-ins)
  2. In the Add-ins dialog, click My add-ins (left sidebar)
  3. Scroll to the bottom → Click + Add a custom add-in → Add from file…
  4. Select your manifest.xml file → Click Open
  5. Click Install on the warning dialog
  6. The add-in is now installed. Close the dialog.
  7. Open a new email — the My Templates button appears in the ribbon.

Outlook Web App (OWA — outlook.office.com):
  1. Compose a new email → click the ⋯ More apps button in the compose toolbar
  2. Click Get Add-ins
  3. Click My add-ins → + Add a custom add-in → Add from URL
  4. Enter the direct URL to your manifest.xml, e.g. https://your-github.io/my-templates-addin/manifest.xml
  5. Click OK → Install

🔘 Enable / Disable (Like Adobe PDF Add-in)

Users can enable or disable the add-in from Outlook's Manage Add-ins page — the same place where Adobe Acrobat, Grammarly, and other add-ins are managed:

  1. In Outlook: File → Manage Add-ins (or Get Add-ins → My add-ins)
  2. Find My Email Templates in the list
  3. Toggle the checkbox to enable or disable it
  4. Or click the ⋯ menu next to it → Remove to uninstall
ℹ️ If deployed via M365 Admin Center, individual users can still disable it for themselves, but admins control whether it can be fully removed.

💾 How Templates Are Stored

Templates are stored using OfficeRuntime.storage — Microsoft's persistent, per-user storage API for Office add-ins. This is:

  • Stored locally on the user's machine, tied to their Windows/Office profile
  • Persists across Outlook restarts and Windows reboots
  • Not deleted when the add-in is disabled (only when uninstalled + data cleared)
  • Up to 10 MB of template data per user (thousands of templates)
  • Export feature available inside the add-in (ℹ️ button → Export All Templates)
💡 Unlike the VBA approach, this storage is managed by Office and does not require AppData folder access or file permissions.
